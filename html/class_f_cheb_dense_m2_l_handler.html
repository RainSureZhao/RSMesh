This class precomputes the M2L operators for all ( possible interacting cells in the far-field) interactions for the Chebyshev interpolation approach.

PB: FChebDenseM2LHandler does not seem to support non_homogeneous kernels! In fact nothing appears to handle this here (i.e., adapt scaling and storage to MatrixKernelClass::Type). Given the relatively important cost of the Chebyshev variant, it is probably a choice not to have implemented this feature here but instead in the ChebyshevSym variant. But what if the kernel is non homogeneous and non symmetric (e.g. Dislocations)...

TODO Specialize class (see UnifM2LHandler) OR prevent from using this class with non homogeneous kernels ?!
