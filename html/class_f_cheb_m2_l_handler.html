This class precomputes and compresses the M2L operators for all ( possible interacting cells in the far-field) interactions for the Chebyshev interpolation approach. The class uses the compression via a truncated SVD and represents the compressed M2L operator as with . The truncation rank is denoted by and is determined by the prescribed accuracy . Hence, the originally of size times for all interactions is reduced to only one and one , each of size , and , each of size .

PB: FChebM2LHandler does not seem to support non-homogeneous kernels! In fact nothing appears to handle this here (i.e. adapt scaling and storage to MatrixKernelClass::Type). Given the relatively important cost of the Chebyshev variant, it is probably a choice not to have implemented this feature here but instead in the ChebyshevSym variant. But what if the kernel is non homogeneous and non symmetric (e.g. Dislocations)...

TODO Specialize class (see UnifM2LHandler) OR prevent from using this class with non homogeneous kernels ?!
