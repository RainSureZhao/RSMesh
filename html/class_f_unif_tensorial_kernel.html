class FUnifTensorialKernel< FReal, CellClass, ContainerClass, MatrixKernelClass, ORDER, NVALS >

作者
Pierre Blanchard (pierr.nosp@m.e.bl.nosp@m.ancha.nosp@m.rd@i.nosp@m.nria..nosp@m.fr)

This kernels implement the Lagrange interpolation based FMM operators. It implements all interfaces (P2P,P2M,M2M,M2L,L2L,L2P) which are required by the FFmmAlgorithm and FFmmAlgorithmThread.

PB: 3 IMPORTANT remarks !!!

1) Handling tensorial kernels (DIM,NRHS,NLHS) and having multiple rhs (NVALS) are considered 2 distinct features and are currently combined.

2) When it comes to applying M2L it is NOT much faster to loop over NRHSxNLHS inside applyM2L (at least for the Lagrange case). 2-bis) During precomputation the tensorial matrix kernels are evaluated blockwise, but this is not always possible. In fact, in the ChebyshevSym variant the matrix kernel needs to be evaluated compo-by-compo since we currently use a scalar ACA.

3) We currently use multiple 1D FFT instead of multidim FFT since embedding is circulant. Multidim FFT could be used if embedding were block circulant. TODO investigate possibility of block circulant embedding
