This class precomputes the M2L operators $[K_1,\dots,K_{316}]$ for all ( $7^3-3^3 = 316$ possible interacting cells in the far-\/field) interactions for the Chebyshev interpolation approach.

PB\+: \doxylink{class_f_cheb_dense_m2_l_handler}{FCheb\+Dense\+M2\+LHandler} does not seem to support non\+\_\+homogeneous kernels! In fact nothing appears to handle this here (i.\+e., adapt scaling and storage to Matrix\+Kernel\+Class\+::\+Type). Given the relatively important cost of the Chebyshev variant, it is probably a choice not to have implemented this feature here but instead in the Chebyshev\+Sym variant. But what if the kernel is non homogeneous and non symmetric (e.\+g. Dislocations)...

TODO Specialize class (see Unif\+M2\+LHandler) OR prevent from using this class with non homogeneous kernels ?!
